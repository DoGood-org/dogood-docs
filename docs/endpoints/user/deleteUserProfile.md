# DELETE /user/profile

## Призначення

Видалення користувача.

## Авторизація

✅ Потрібна

## Додаткова логіка
- якщо користувач є host:
    - видаляються його tasks
- після видалення виконується cleanup orphan locations

## Що робить бекенд
1. Перевіряє авторизацію
2. Шукає host запис користувача
3. Видаляє tasks користувача як host
4. Видаляє user
5. Видаляє orphan locations

## Що пишеться в БД
- видаляється запис User
- видаляються пов’язані Task (якщо користувач host)

## Response
```json
{
  "status": "success",
  "code": "USER_DELETED"
}
```

## ❌ Помилки
| Код               | HTTP | Опис             |
| ----------------- | ---- | ---------------- |
| AUTH_UNAUTHORIZED | 401  | Не авторизований |
