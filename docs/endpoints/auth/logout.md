# POST /auth/logout

## Призначення

Вихід користувача з системи.

## Авторизація

❌ Не потрібна (accessToken)

⚠️ Потрібен refreshToken у cookies (встановлюється після login)


## Request body

❌ Відсутній

## 🍪 Cookies (вхід)
| Cookie       | Обов'язкове | Опис                        |
| ------------ | ----------- | --------------------------- |
| refreshToken | так         | Використовується для logout |

## 🔥 Додаткова логіка
- refreshToken береться з cookies
- токен декодується
- всі refresh токени користувача видаляються
- cookies очищаються

## Що робить бекенд
1. Отримує refreshToken з cookies
2. Перевіряє його валідність
3. Витягує userId
4. ❗ Видаляє ВСІ refresh токени користувача
5. Очищає cookies
6. Повертає відповідь

## Що пишеться в БД
- RefreshToken → видаляються всі записи користувача

## Response
```
204 No Content
```

👉 body відсутній

## 🍪 Cookies (після logout)
| Cookie       | Стан        |
| ------------ | ----------- |
| accessToken  | ❌ видалений |
| refreshToken | ❌ видалений |

## ⚠️ Важливо
- logout завершує ВСІ сесії користувача
- користувач буде розлогінений на всіх пристроях

## ❌ Можливі помилки
| Код                        | HTTP | Опис                    |
| -------------------------- | ---- | ----------------------- |
| AUTH_REFRESH_TOKEN_INVALID | 400  | refresh token відсутній |

## Важливі нюанси
- якщо refreshToken відсутній → помилка
- logout НЕ є idempotent (на відміну від багатьох API)
- accessToken не використовується

## Пов’язані сутності

RefreshToken
